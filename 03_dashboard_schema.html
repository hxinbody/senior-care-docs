AI 전화통화 데이터 (신규)
S1~S6 시나리오에서 수집되는 모든 데이터의 저장 구조. 통화 세션을 중심으로 하위 레코드가 연결됩니다.
AICall_Session NEW TABLE
필드명타입설명
SessionID PK BIGINT IDENTITY 통화 세션 고유 ID
UID FK IDX BIGINT 사용자 ID ← Profile_UserInfo.UID
ScenarioType VARCHAR(20) 'S1_CHECKIN' | 'S2_REPORT' | 'S3_COACHING' | 'S4_COGNITIVE' | 'S5_FRAILTY' | 'S6_EMERGENCY'
CallStartTime DATETIME2 통화 시작 시각 (JST)
CallEndTime DATETIME2 통화 종료 시각

CallStatus VARCHAR(20) 'COMPLETED' | 'NO_ANSWER' | 'BUSY' | 'DROPPED' | 'ESCALATED'
CallDirection VARCHAR(10) 'OUTBOUND' (정기) | 'INBOUND' (사용자 요청) | 'CALLBACK'
TranscriptJSON NULL NVARCHAR(MAX) 대화 전문 (JSON). 의료기관 공유 시 요약본만 노출
SummaryText NULL NVARCHAR(2000) LLM 생성 통화 요약 (의료진 열람용). 3~5문장

GET /medical/patients
담당 환자 목록 (Dashboard_PatientList 뷰). OverallRiskLevel, FrailtyLevel로 필터/정렬 가능.
Response: StandardResponse[List[PatientListItem]]
GET /medical/patients/{uid}/detail
환자 상세 (Dashboard_PatientDetail 뷰). 체성분 시계열 + AI 전화 이력 + 위험도 종합.
Response: StandardResponse[PatientDetailResponse]
GET /medical/patients/{uid}/bodycomp?from=&to=
체성분 시계열 데이터. 기존 Measure_BodyComposition 재사용.
Response: StandardResponse[List[BodycompDto]] ← 기존 DTO 재사용
GET /medical/patients/{uid}/calls?from=&to=
AI 전화 통화 이력. SummaryText만 노출 (TranscriptJSON은 DataScope 검증 후).
Response: StandardResponse[List[CallSessionSummary]]
GET /medical/patients/{uid}/frailty-history
프레일티 체크 이력 + InBody 교차 검증 데이터 포함.
Response: StandardResponse[List[FrailtyCheckResponse]]
GET /medical/patients/{uid}/cognitive-history
인지기능 스크리닝 이력 + 추이 분석.
Response: StandardResponse[List[CognitiveScreenResponse]]
GET /medical/alerts?level=&acknowledged=false
알림 피드 (Dashboard_AlertFeed 뷰). 미확인 알림 우선 정렬.
Response: StandardResponse[List[AlertFeedItem]]
